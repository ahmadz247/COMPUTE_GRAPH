Computational Graph Web Application

Project Overview

This system implements a web-based computational graph visualization and management tool. It combines publish-subscribe architecture, HTTP server implementation, and interactive web visualization to create a comprehensive distributed computing demonstration.

Architecture

The system is organized into five main packages:

Core Packages

  • {@link graph} - Core publish-subscribe system with Topics, Agents, and Messages
  • {@link configs} - Configuration management and graph representation with cycle detection
  • {@link server} - Multi-threaded HTTP server implementation
  • {@link servlets} - Request handlers for various endpoints
  • {@link views} - HTML generation and graph visualization

Key Design Patterns

  • Singleton Pattern - {@link graph.TopicManagerSingleton} ensures single topic registry
  • Decorator Pattern - {@link graph.ParallelAgent} adds async processing to any agent
  • Observer Pattern - {@link graph.Topic} implements publish-subscribe messaging
  • Factory Pattern - {@link configs.GenericConfig} creates agents via reflection
  • Strategy Pattern - Different layout algorithms in {@link views.HtmlGraphWriter}

SOLID Principles

The codebase demonstrates all five SOLID principles:

  • Single Responsibility - Each class has one clear purpose
  • Open/Closed - New agent types can be added without modifying core classes
  • Liskov Substitution - All agents are interchangeable via the Agent interface
  • Interface Segregation - Clean, minimal interfaces throughout
  • Dependency Inversion - Depend on abstractions (interfaces) not implementations

Key Algorithms

  • Cycle Detection - DFS with white-gray-black coloring in {@link configs.Graph}
  • Graph Layout - Multiple algorithms (bipartite, circular, grid) in {@link views.HtmlGraphWriter}
  • Longest Prefix Matching - Servlet routing in {@link server.MyHTTPServer}
